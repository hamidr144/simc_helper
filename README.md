# Simcraft Helper Suite

A collection of Python scripts and a Web GUI to automate SimulationCraft (SimC) workflows, from generating item combinations to running multi-stage simulations with real-time progress tracking.

## Features

- **Web GUI:** A clean, browser-based interface to manage your engine, select gear combinations, run simulations, and view the final report.
- **Smart Reporting:** The web interface highlights gear differences compared to your baseline and integrates Wowhead tooltips on hover for quick item comparisons.
- **Automated Engine Management:** Easily clone, update, and build the SimulationCraft engine directly from the source (`~/.simc`).
- **3-Stage Refinement:** Performs multi-stage simulations (100 -> 2000 -> Final iterations) to quickly filter out poor combinations, saving massive CPU time.
- **OOM-Safe Batching:** Automatically splits large combo datasets (>400 profiles) into memory-safe batches to prevent SimulationCraft from crashing your system. Uses global sorting to guarantee the absolute top performers are kept across all batches.

## Setup & Installation

### Requirements
- Python 3.6+
- Git, `make`, and a C++ compiler (for building SimC)
- A SimC addon export from World of Warcraft.

### Web Server
To launch the Web GUI, install the required packages and start the server from the project root:

```bash
pip install -r requirements.txt
python3 -m src.web.main
```
Then navigate to `http://localhost:8000` in your browser.

## Project Structure

- `src/web/`: Contains the FastAPI backend and Vanilla CSS frontend.
- `src/cli/`: Standalone Python scripts for input generation and simulation management.
- `scripts/`: Shell utilities for environment management (e.g., building SimC).

## Using the Web GUI (Recommended)

1. **Setup & Addon:** Update/build the SimC engine (installs to `~/.simc`) and paste your `/simc` addon output.
2. **Gear Selection:** Choose the items from your bags you wish to test in different combinations. The tool automatically pre-selects your currently equipped items.
3. **Run Simulation:** Watch the live console output as the 3-stage refinement process executes.
4. **Final Report:** View a ranked list of combinations. Gear differences from your baseline are highlighted in green, and all items feature hoverable Wowhead tooltips. You can also view the full HTML report generated by SimC.

## Command-Line Scripts

If you prefer the terminal, you can use the standalone scripts directly:

### 1. `scripts/manage_simc.sh`
Automatically clones (or pulls) the SimC repository to `~/.simc` and builds the optimized engine using all available CPU cores.

### 2. `src/cli/generate_input.py`
An interactive script to generate a `.simc` input file by combining your character's base profile with selected gear combinations, enchantments, and gems. Uses `config.json` to define testable enchant/gem IDs.

**Usage:**
```bash
python3 src/cli/generate_input.py
```

### 3. `src/cli/sim_helper.py`
A robust wrapper for running SimulationCraft with real-time progress updates, OOM-safe batching, and automated report management.

**Usage:**
Run the script providing the path to your `simc` executable and your input file:
```bash
python3 src/cli/sim_helper.py simc_path=~/.simc/engine/simc input_file=generated_sim.simc [stage1_percent_best=25%] [stage2_percent_best=25%] [start_server=1]
```

**Arguments:**
- `simc_path`: Path to the SimulationCraft executable.
- `input_file`: Path to the `.simc` file containing profiles.
- `stage1_percent_best`: % or number of profiles to keep after Stage 1 (Default: Dynamic).
- `stage2_percent_best`: % or number of profiles to keep after Stage 2 (Default: Dynamic).
- `start_server`: Set to `1` to automatically start a local HTTP server and open the final report.

## Configuration (`config.json`)
The `config.json` file allows you to specify lists of IDs to test for each slot when using `generate_input.py`. 

Example:
```json
{
    "enchantments": {
        "head": [7991, 8015],
        "chest": [7956, 7957]
    },
    "gems": {
        "neck": [240890, 240892]
    }
}
```
